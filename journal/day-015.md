## 💻 Project of the Day
- **Name:** Coffee Machine Simulation
- **Description:**  
A command-line simulation of a coffee machine that allows users to order drinks (espresso, latte, cappuccino), process coin payments, check available resources, and generate reports. The machine validates resources before processing orders and tracks profit over time.

## 🤔 Challenges & Solutions
- **Challenge:** Managing multiple responsibilities (resources, money, user input).  
  **Solution:** Broke the program into focused functions like `is_resource_sufficient()`, `process_coins()`, and `make_coffee()`.

- **Challenge:** Keeping track of machine state across multiple orders.  
  **Solution:** Used global variables (`resources`, `profit`) to persist state throughout execution.

- **Challenge:** Handling user input and flow control.  
  **Solution:** Implemented a main loop with conditions for `off`, `report`, and drink selection.
